---
name: Workflow — always review before deploy
description: Never commit and push blog posts without user reviewing the content first
type: feedback
---

Always let David read and review blog post content before committing and pushing. Write the post, show it locally, wait for approval.

**Why:** Blog posts represent David's voice and perspective. Pushing without review risks publishing content that doesn't match his intent.

**How to apply:** After creating/editing a blog post, tell the user to review it at localhost. Only commit+push when explicitly asked.
